# Crypto Price Tracker

## Описание проекта

Проект представляет собой сервис для получения и хранения цен криптовалют BTC и ETH с биржи Deribit. Основная цель — разработка backend-API для работы с историческими данными по ценам и предоставление актуальной информации через REST API.

Проект реализован с использованием **FastAPI** и **PostgreSQL**. Для периодического получения данных с биржи используется **Celery**.

---

## Функционал

### Клиент Deribit
- Каждую минуту получает текущую цену BTC/USD и ETH/USD (index price) через API биржи Deribit.
- Сохраняет данные в PostgreSQL: тикер валюты, текущую цену и UNIX timestamp.

### API на FastAPI
- Все методы используют **GET-запросы** и имеют обязательный query-параметр `ticker`.
- Эндпоинты:
  - `/prices/all` — получение всех сохранённых данных по указанной валюте.
  - `/prices/latest` — получение последней цены указанной валюты.
  - `/prices/by-date` — получение цен указанной валюты с фильтром по дате (`from_date` и `to_date`).

## Design Decisions

### Архитектура
- **Слой репозиториев** (`BaseRepository`) отвечает только за работу с базой данных через SQLAlchemy ORM.  
- **Слой сервисов** (`PriceService`) инкапсулирует бизнес-логику: получение данных из репозиториев и взаимодействие с внешними API (Deribit).  
- **FastAPI роутеры** используют сервисы для обработки запросов и подготовки ответов клиенту.  

Такое разделение ответственности упрощает поддержку, тестирование и масштабирование.

### Работа с внешним API
- Асинхронный клиент на базе `aiohttp` (`DeribitClient`) получает текущие цены тикеров `btc_usd` и `eth_usd`.  
- Периодическая задача на **Celery** запускается каждую минуту.  
- В текущей реализации ошибки HTTP-запросов (тайм-ауты, недоступность API) не обрабатываются, но планируется логирование и retry в будущем.

### База данных
- Используется **PostgreSQL** с таблицей `ticker_prices`:

| Поле       | Тип       | Описание                       |
|------------|-----------|--------------------------------|
| `id`       | BigInteger | Первичный ключ, автогенерация |
| `ticker`   | String     | Название тикера (`btc_usd`, `eth_usd`) |
| `price`    | Float      | Цена валюты                   |
| `timestamp`| BigInteger | UNIX timestamp времени записи |

- Возможна установка индексов по `ticker` и `timestamp` для ускорения запросов по истории.

### Асинхронность и Celery
- Все запросы к базе и к внешнему API выполняются **асинхронно**.  
- Celery задачи также асинхронные и выполняются каждую минуту, обеспечивая актуальность данных без блокировки API.

### Валидация и сериализация
- Используются **Pydantic модели** (`TickerPriceAdd`, `TickerPriceRead`) с `from_attributes=True` для сериализации ORM объектов.  
- Входные данные валидируются через Enum `Ticker`, что обеспечивает ответ **422** при некорректном тикере.  
- Методы возвращают либо список объектов (`get_all`, `get_by_date`), либо один объект (`get_latest`) — корректно сериализуемый FastAPI.

### Обработка ошибок
- HTTP-исключения обрабатываются через FastAPI:  
- **422 Unprocessable Entity** — некорректный тикер  
- **404 Not Found** — если данные по тикеру или периоду отсутствуют  
- **500 Internal Server Error** — непредвиденные ошибки  
- Глобальные переменные в проекте **не используются**, что обеспечивает чистоту и безопасность кода.

### Контейнеризация и Docker

- Проект подготовлен для работы в **Docker-контейнерах**, включая PostgreSQL, Redis, FastAPI приложение и Celery воркеры.  
- Docker обеспечивает:
  - Изоляцию зависимостей и среды выполнения.
  - Простое масштабирование и переносимость проекта.
  - Возможность легко запускать Celery и базу данных без ручной настройки.
- Используется `docker-compose` для организации взаимодействия контейнеров (PostgreSQL → Redis → приложение → Celery).  
- Volumes обеспечивают сохранение данных при перезапуске контейнеров (`postgres_data`, `redis_data`).
## Развёртывание проекта

### 1. Клонирование репозитория
```bash
git clone <[https://gitlab.com/cenat47-group/crypto-price-tracker.git](https://gitlab.com/cenat47-group/crypto-price-tracker.git)>
cd crypto-price-tracker
```
2. Настройка переменных окружения
Скопируйте шаблон .env.example в .env и при необходимости измените значения:

```bash
cp .env.example .env
```
Пример содержимого .env:

MODE=PRODUCTION 

DB_HOST=localhost
DB_PORT=5432
DB_USER=user
DB_PASS=pass
DB_NAME=crypto_db

REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASS=user
3. Сборка и запуск контейнеров
Используем Docker Compose для поднятия всех сервисов:

```bash

docker-compose up --build
```
Контейнеры:

PostgreSQL (postgres)

Redis (redis)

FastAPI приложение (app) — автоматически выполняет миграции при старте.

Celery воркер и Celery Beat (celery, celery_beat)

Все зависимости и окружение настраиваются через .env.

4. Проверка работы
API доступно на http://localhost:8000

Swagger документация: http://localhost:8000/docs

Проверка базы данных:

```bash
docker exec -it crypto_postgres psql -U $DB_USER -d $DB_NAME
```
5. Остановка и удаление контейнеров
Остановить контейнеры:

```bash
docker-compose down
```
Удалить контейнеры и volumes с данными:

```bash
docker-compose down -v
```
